# Hackathon Brief

## 命题

面向医学课程建设，把 7 本分散教材整合为一份可解释、可复核、可问答的精华知识体系，目标压缩到原始体量的 30% 左右，同时保留教学逻辑链路。

## 目标用户

医学基础课/整合课程教师、课程负责人，以及需要跨学科学习路径的医学生。

## 评审标准

- 文档完整：需求分析、系统设计、Agent 架构、README、整合报告均可查阅。
- P0 功能闭环：教材导入、章节拆分、知识图谱、跨教材整合决策、RAG 问答、教师反馈修改、报告生成。
- 可解释性：每条整合决策包含理由、置信度、影响节点和节省字符量。
- 工程可运行：本地和 Docker 均能启动，缺少外部 API 时有降级路径。

## MVP 输入

7 本内置医学教材 Markdown，外加用户上传的 PDF/DOCX/MD/TXT 教材。

## MVP 输出

跨教材知识图谱、整合决策列表、带引用的问答结果、可提交的 Markdown 整合报告。

## 演示脚本

1. 用一句话说明用户痛点。
2. 输入一组真实或拟真的样例数据。
3. 展示 AI 生成结果。
4. 点出结果如何节省时间、降低成本或提升质量。
5. 说明下一步可扩展能力。
