# 接口文档

## 1. 基础信息

- 默认地址：`http://localhost:7860`
- API 前缀：`/api`
- 数据格式：JSON
- 文件上传：`multipart/form-data`

## 2. 健康检查

### GET `/health`

返回服务状态、外部服务配置状态和数据统计。

```json
{
  "ok": true,
  "service": "knowledge-integrator",
  "config": {
    "deepseek": {
      "configured": true,
      "model": "deepseek-v4-flash"
    }
  }
}
```

## 3. 教材管理

### POST `/api/upload`

上传单个教材文件，支持 `.pdf`、`.docx`、`.md`、`.markdown`、`.txt`。

请求：

```bash
curl -F "file=@教材.pdf" http://localhost:7860/api/upload
```

响应：

```json
{
  "textbook": {
    "id": "user_xxx",
    "title": "教材",
    "filename": "教材.pdf",
    "status": "ready",
    "total_chars": 385000,
    "chapter_count": 42
  },
  "graph_stats": {
    "node_count": 42,
    "edge_count": 72
  },
  "rag_index": {
    "status": "indexed",
    "chunks": 300
  }
}
```

### GET `/api/textbooks`

返回教材列表。

### GET `/api/textbooks/{textbook_id}`

返回教材详情和章节列表。

### DELETE `/api/textbooks/{textbook_id}`

删除指定教材，并重建整合图谱和本地 RAG 索引。

## 4. 知识图谱

### POST `/api/knowledge/extract/{textbook_id}`

重新构建单本教材图谱。`use_llm=true` 时会尝试调用 DeepSeek 审核候选节点。

### GET `/api/knowledge/graph/{textbook_id}`

返回单本教材知识图谱。

```json
{
  "nodes": [
    {
      "id": "node_xxx",
      "name": "动作电位",
      "definition": "...",
      "category": "核心概念",
      "body_system": "神经系统",
      "frequency": 1,
      "source_textbooks": ["生理学"]
    }
  ],
  "edges": [
    {
      "id": "edge_xxx",
      "source": "node_a",
      "target": "node_b",
      "relation_type": "contains",
      "description": "章节结构显示上级主题包含该知识点。"
    }
  ],
  "stats": {
    "node_count": 79,
    "edge_count": 120
  }
}
```

### GET `/api/knowledge/graph/merged`

返回跨教材整合图谱。

## 5. 跨教材整合

### POST `/api/integrate/start`

重新执行跨教材整合。

### GET `/api/integrate/status`

返回整合任务状态和统计。

### GET `/api/integrate/stats`

返回压缩比、节点数、决策数、RAG chunk 数。

```json
{
  "source_chars": 8800495,
  "merged_chars": 160312,
  "compression_ratio": 0.0182,
  "target_ratio": 0.3,
  "target_met": true,
  "raw_node_count": 795,
  "merged_node_count": 772,
  "decision_count": 15,
  "rag_chunks": 5661
}
```

### GET `/api/integrate/decisions`

返回整合决策列表。

### PUT `/api/integrate/decisions/{decision_id}`

更新决策状态或理由。

请求：

```json
{
  "status": "rejected",
  "reason": "教师认为这两个概念不应合并"
}
```

状态可选：`active`、`hidden`、`rejected`。

## 6. RAG 问答

### POST `/api/rag/index`

生成本地 chunk，并在 Dify 配置完整时同步知识库。

查询参数：

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| sync_dify | true | 是否同步 Dify |
| sync_all | true | 是否同步全部 chunk |
| limit | null | 限制同步条数 |
| batch_size | 5000 | Dify 单批 chunk 数 |

### GET `/api/rag/status`

返回索引状态。

### POST `/api/rag/query`

请求：

```json
{
  "question": "肺炎链球菌的荚膜有什么作用？",
  "conversation_id": "optional"
}
```

响应：

```json
{
  "answer": "荚膜可帮助肺炎链球菌抵抗吞噬...",
  "citations": [
    {
      "textbook": "医学微生物学",
      "chapter": "第十二章 细菌",
      "relevance_score": 0.92,
      "chunk_preview": "..."
    }
  ],
  "conversation_id": "optional",
  "provider": "dify"
}
```

provider 可能值：

| 值 | 含义 |
| --- | --- |
| `dify` | 使用 Dify Chat API 成功回答 |
| `dify_error_fallback` | Dify 调用失败，回退本地 chunk 检索 |
| `local_waiting_dify` | Dify 未配置，使用本地检索 |

## 7. 教师对话

### POST `/api/chat/message`

请求：

```json
{
  "session_id": "default",
  "message": "为什么把炎症相关知识点合并？"
}
```

响应复用 RAG response 结构。系统会识别“为什么合并”“不要合并”“保留”等整合反馈意图。

### GET `/api/chat/history/{session_id}`

返回同一会话内的对话历史。

## 8. 报告与学习材料

### GET `/api/report/generate`

生成 Markdown 报告内容。

### GET `/api/report/download`

以 `text/markdown` 返回报告。

### GET `/api/game/skill-tree`

返回内置闯关学习样例。

### GET `/api/multimodal/status`

返回多模态材料状态。NotebookLM 未配置时使用缓存/降级材料。
