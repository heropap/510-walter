# 学科知识整合智能体 · 需求分析

## 一、赛题理解

本项目面向“多教材知识整合”场景：教师需要把 7 本医学相关教材中的重复、交叉和前后依赖知识整理为一套更紧凑的课程材料。系统不是简单摘要工具，而是要给出可解释的整合路径：哪些知识点被合并、为什么合并、保留了哪些证据、教师如何撤销或调整决策。

当前 MVP 以 7 本内置医学教材为样例，覆盖局部解剖学、组织学与胚胎学、生理学、医学微生物学、病理学、传染病学、病理生理学。目标是把教材内容压缩到原始体量约 30%，即“7 本变成约 2.1 本”的精华学习材料，同时支持 RAG 问答和课堂复核。

## 二、目标用户

- 医学整合课程负责人：需要在短时间内搭建跨学科教学大纲和授课材料。
- 医学基础课教师：需要定位不同教材中对同一概念的重复讲解和差异化侧重点。
- 医学生：需要按器官系统、疾病机制或病原体路径理解跨教材知识链。

## 三、核心痛点

- 多本教材重复描述同一概念，人工去重成本高。
- 同一医学概念在不同学科中粒度不同，例如“肺炎”同时涉及解剖结构、病原体、炎症、气体交换和临床防治。
- 传统摘要难以说明“为什么删、为什么合并”，教师无法快速复核。
- 问答系统如果脱离教材原文，容易产生无法追溯的回答。
- 比赛时间有限，系统必须在外部 API 未配置时仍可演示核心闭环。

## 四、P0 范围

- 教材管理：展示内置教材，支持上传 MarkItDown 可转换格式，包括 Markdown/TXT、PDF、Word、PowerPoint、Excel、CSV/JSON/XML、HTML、EPUB、ZIP、Outlook MSG、图片和音频。
- 文档解析：统一转换为 Markdown，并按章节拆分。
- 知识图谱：基于章节标题、定义句、目录层级和规则标签生成节点与关系。
- 跨教材整合：按标准化标题和来源频次生成合并决策，保留 affected nodes、result node、reason、confidence、char_saved。
- 教师反馈：教师可把决策标记为 active、hidden、rejected；其中 rejected 必须刷新整合图谱并恢复来源节点。
- RAG 问答：优先调用 Dify；未配置时回退到本地 chunk 检索，并展示来源章节片段。
- 整合报告：生成并保存 Markdown 报告，包含数据概览、教材清单、主要决策、教学建议和已知局限。

## 五、输入与输出

输入：

- 内置 7 本医学教材 Markdown。
- 用户上传教材文件：MarkItDown 支持的 PDF、Office、CSV/JSON/XML、HTML、EPUB、ZIP、MSG、图片、音频，以及 Markdown/TXT。
- 教师反馈：整合面板按钮或对话中的“不要合并/分开/不应该合并”等意图。

输出：

- 单本教材图谱与跨教材整合图谱。
- 整合决策列表和教师修改后的决策状态。
- RAG 问答结果与引用片段。
- `report/整合报告.md` 交付文件。

## 六、关键业务规则

- Markdown 是唯一中间格式。上传文件先转 Markdown，再拆章节，避免图谱、RAG、报告使用不同数据源。
- RAG 分块使用 500-800 字主窗口和约 100 字重叠，尽量覆盖一个完整知识点，并降低知识点被截断的概率。
- 每个 chunk 注入教材名和章节名，确保 Dify 或本地检索返回时具备来源信息。
- 整合决策必须可解释：至少包含动作、影响节点、结果节点、理由、置信度和节省字符数。
- 教师拒绝合并后，系统应重新生成整合图谱：原本被合并的来源节点分别保留，统计数据随之变化。
- 外部服务降级不阻塞 P0：DeepSeek、Dify、NotebookLM 未配置时，系统仍可展示本地规则图谱、本地检索和缓存学习材料。

## 七、验收标准

- 打开首页能看到 7 本教材、整合图谱和压缩统计。
- 点击“重新整合”能刷新图谱、统计和整合决策。
- 在决策卡片点击“撤销”后，该决策状态变为 rejected，整合图谱节点数发生相应变化，刷新后仍保持教师反馈。
- RAG Tab 能返回教材章节引用；Dify 未配置时明确显示本地检索降级。
- 调用 `/api/report/generate` 后，`report/整合报告.md` 存在且包含已知局限章节。

## 八、已知局限与边界

- 当前图谱主要依赖章节结构和规则抽取，医学同义词、英文缩写、跨层级机制关系仍需要 LLM 或教师复核增强。
- PDF、Office、HTML、图片/音频等复杂格式的解析质量取决于源文件版式和可选依赖；扫描版 PDF 需要 OCR 兜底。
- 压缩比是字符和节点规模估算，不能直接等价于教学质量；正式课程仍需教师按教学目标复核。
- Dify 未配置时，本地检索只能提供片段级依据，不具备完整向量 + BM25 混合检索能力。
- NotebookLM 和闯关游戏是创新展示层，不作为 P0 核心依赖。
