## 重点案例：肺炎链球菌跨教材整合

肺炎链球菌相关内容在 7 本教材中形成一条完整教学链路：医学微生物学解释荚膜、毒力和免疫逃逸；局部解剖学提供肺叶、支气管和胸膜腔位置框架；组织学与胚胎学补充肺泡隔、肺泡上皮和毛细血管屏障；生理学说明通气/血流比例与气体交换；病理学呈现肺泡腔渗出、实变和炎症细胞浸润；病理生理学解释低氧血症、呼吸衰竭和代偿；传染病学落到传播、诊断、抗菌药物和预防。整合后的讲义不是把 7 本书简单拼接，而是按“病原体入侵 → 解剖定位 → 组织屏障 → 功能受损 → 病理改变 → 病理生理后果 → 临床处理”的顺序重排，保留关键证据来源并删除重复背景段落。

该案例也对应独立的“细菌之旅”互动叙事游戏：每个站点切换一本教材的观察视角，让学生用同一个临床故事理解跨学科知识如何串联。它可作为整合报告之外的课堂活动材料，帮助评委看到项目不仅能生成图谱和 RAG，也能把整合结果转化为可用的教学体验。

## 教学使用建议

- 先用医学导航按器官系统筛选，再进入图谱查看跨教材来源。
- 对高频节点优先生成课堂讲解材料，低置信度合并项由教师复核。
- RAG 问答必须展示教材和章节引用，避免脱离教材自由发挥。

## 已知局限与复核项

- PDF/DOCX 转 Markdown 依赖原文件版式质量；扫描版或复杂表格需要 OCR/人工校对兜底。
- 当前语义整合以章节标题、定义句和来源频次为主，医学同义词、缩写和跨层级关系仍需要教师复核。
- Dify 未配置时系统使用本地关键词 + BM25 混合检索降级，能展示引用片段；完整生成链路仍建议配置 Dify。
- 30% 压缩目标按字符和节点规模估算，最终教学完整性需要结合课程大纲和课堂目标再次确认。
- NotebookLM 和闯关游戏属于可选创新层；核心 P0 不依赖这些第三方或样例输出。
